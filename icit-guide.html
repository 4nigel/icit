Overview

Offline single-file 3D viewer for IFC, OBJ, STL, PLY, and glTF/glb models. No install, no internet, no data leaves your machine — drop the HTML file anywhere and open it in a browser.

Formats: IFC4 · OBJ · STL · PLY · glTF/glb

FOSS stack: three.js (MIT) + web-ifc (MPL-2.0)

QTO Workflow
  1. Load your IFC file
  2. Click any element of the class you want (e.g. a window)
  3. Press L — selects all of that IfcClass
  4. Or Ctrl+click the class label in the tree to add/remove whole classes
  5. Click 📤 CSV — exports Psets and Quantities for the full selection

Multiple classes: select elements from several classes, then L expands each to the full class. Or Ctrl+click multiple class labels in the tree.

Section Planes

Three dual-slider section planes — X (left/right), Y (depth), Z (height/plan). Enable each axis with its toggle button.

Both sliders on one axis clip from opposite sides, letting you isolate a floor slab, a zone, or a single storey by dragging inward from each end.

Section ranges are reset automatically each time a model loads.

⚠ QTO Accuracy — Read Before Using Quantities for Cost Planning

ICit is a quantity reader, not a quantity measurer. It does not interrogate mesh geometry to calculate dimensions. Every value in the CSV — including CALC columns — is derived arithmetically from data already in the IFC file, authored by the BIM designer.

This means ICit cannot detect modelling errors, cannot independently verify the BIM author's numbers, and cannot produce quantities that are certifiable without reference to the source BIM model.

What Pset columns are

Values under Pset_WallCommon.Height (m) and similar columns are read directly from IfcElementQuantity entities that the BIM authoring tool wrote into the file. Their accuracy depends entirely on the BIM author's modelling discipline, the Level of Development of the model, and the export settings used. ICit passes these through without modification.

What CALC columns are

CALC.Perimeter (m) is currently the only viewer-derived quantity. It is computed as (2×Height) + (2×Width) using the BIM-authored Height and Width values when the exporter did not supply a perimeter directly. It is only geometrically correct for rectangular elements. It is wrong for L-shaped walls, tapered elements, irregular profiles, and any element where width varies. If the BIM author's H or W is wrong, CALC.Perimeter is wrong.

Recommended use

Use ICit CSV output for preliminary review, cross-checking, and coordination — not for certified cost plans or contract documentation. For certified QTO, process the IFC file through a dedicated QTO tool (CostX, Buildsoft, or similar) that performs auditable geometric analysis.

A discrepancy between a Pset_*.Area (m2) value and what you'd expect from the model is a flag to raise with the BIM author — not something ICit can resolve.
